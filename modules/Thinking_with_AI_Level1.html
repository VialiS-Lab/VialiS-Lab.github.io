📐 QHER 프레임워크 v3 (최종 정의)

QHER는 Question → Hypothesis → Evidence → Reasoning의 약자입니다.
Decision과 Synthesis는 Reasoning 단계의 내부 프로세스로 통합되어, 외부에서는 항상 Q-H-E-R-Reflection이라는 일관된 4+1 단계를 유지합니다.

```
┌─────────────────────────────────────────────────────────────────┐
│                    QHER ENGINE v3.0                             │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ┌───────────────────────────────────────────────────────────┐ │
│  │  Q. Question (질문 설계)                                  │ │
│  │  → 출력: 구조화된 질문 (범위·관점·기준·형식 포함)         │ │
│  └───────────────────────┬───────────────────────────────────┘ │
│                          ▼ (Auto-Rule 1: 질문 유형 분석)       │
│  ┌───────────────────────────────────────────────────────────┐ │
│  │  H. Hypothesis (가설 설계)                                │ │
│  │  → 출력: "만약 A라면 B이다" 형식의 검증 가능한 가설        │ │
│  └───────────────────────┬───────────────────────────────────┘ │
│                          ▼ (Auto-Rule 2: 가설 기반 증거 수집)  │
│  ┌───────────────────────────────────────────────────────────┐ │
│  │  E. Evidence (증거 평가)                                  │ │
│  │  → 출력: EQI 점수가 매겨진 증거 목록 (지지/반박 분류)     │ │
│  └───────────────────────┬───────────────────────────────────┘ │
│                          ▼ (Auto-Rule 3: 증거 → 논리 연결)     │
│  ┌───────────────────────────────────────────────────────────┐ │
│  │  R. Reasoning (추론 · 판단 · 종합)                       │ │
│  │  ├─ ① Inference: 증거를 논리로 연결                       │ │
│  │  ├─ ② Decision: 불확실성 하에서 판단 (Expected Value 등)  │ │
│  │  └─ ③ Synthesis: 상충 의견을 통합한 새로운 모델 도출      │ │
│  │  → 출력: 최종 결론 + Confidence Score (1~10)              │ │
│  └───────────────────────┬───────────────────────────────────┘ │
│                          ▼ (Auto-Rule 4: Reflection 트리거)    │
│  ┌───────────────────────────────────────────────────────────┐ │
│  │  🔄 Reflection (되돌아보기)                               │ │
│  │  → 출력: 개선된 질문(Q) → 다음 사이클로 연결              │ │
│  └───────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

🛠️ v3 신규 도구 1: Evidence Quality Index (EQI)

EQI는 증거의 품질을 4가지 기준으로 평가하여 0~5점의 단일 지수로 환산합니다.

EQI 계산 공식

```
EQI = (Source_Reliability × 0.40) + (Directness × 0.30) + (Sample_Size × 0.20) + (Reproducibility × 0.10)
```

각 항목 평가 기준 (1~5점)

항목1점3점5점
출처 신뢰도개인 블로그 / SNS신문 / 잡지학술지 / 공식 기관 / 피어리뷰
직접성간접 추론 / 루머2차 인용 / 해석원본 데이터 / 직접 관찰 / 1차 출처
표본 규모사례 연구 (n<10)중간 규모 (n=30~100)대규모 (n>500) / 메타분석
재현성재현 불가 / 일회성부분적 재현 가능완전 재현 가능 / 공개 데이터

EQI 해석 가이드

EQI 점수등급의미
4.0 ~ 5.0⭐⭐⭐⭐⭐ 매우 강한 증거결정적 증거로 활용 가능
3.0 ~ 3.9⭐⭐⭐⭐ 강한 증거주요 근거로 활용 가능
2.0 ~ 2.9⭐⭐⭐ 보통 증거보조 근거로만 활용
1.0 ~ 1.9⭐⭐ 약한 증거신중한 검토 필요, 거의 배제
0.0 ~ 0.9⭐ 매우 약한 증거근거로 활용 불가

🛠️ v3 신규 도구 2: Reasoning Audit Checklist

Reasoning Audit는 Module 4에서 사용하는 6단계 추론 감사 도구입니다.

Reasoning Audit 체크리스트

```markdown
# Reasoning Audit 체크리스트 (v1.0)

## 1. 주장 (Claim) 명확성
- [ ] 주장이 모호하지 않고 구체적인가?
- [ ] 주장이 측정/검증 가능한 형태인가?

## 2. 가정 (Assumption) 점검
- [ ] 내 추론에 숨겨진 가정은 무엇인가?
- [ ] 그 가정이 맞지 않으면 추론이 무너지는가? (민감도)

## 3. 증거 매핑 (Evidence Mapping)
- [ ] 각 증거가 어떤 중간 추론을 지지하는가?
- [ ] 증거와 주장 사이의 논리적 거리는 얼마나 되는가? (직접적일수록 좋음)

## 4. 누락된 증거 (Missing Evidence)
- [ ] 이 결론에 도달하기 위해 어떤 증거가 더 필요한가?
- [ ] 그 증거를 찾을 수 있는 방법은 무엇인가?

## 5. 반박 및 대안 (Counterarguments)
- [ ] 이 주장에 대한 가장 강력한 반박은 무엇인가?
- [ ] 그 반박을 수용하면 주장이 어떻게 변하는가?

## 6. 최종 신뢰도 (Confidence Score)
- 1~10점 중 내 추론에 대한 신뢰도는?
- [ ] 1-3: 낮음 (재검토 필요)
- [ ] 4-6: 보통 (추가 증거 검토 권장)
- [ ] 7-10: 높음 (확신 있음)
```

⚙️ v3 신규 도구 3: QHER 파이프라인 자동 생성 규칙 (Auto-Rules)

이 규칙들은 Module 6에서 Engine으로 통합되며, 각 단계의 출력이 다음 단계의 입력으로 자동으로 변환되는 방식을 정의합니다.

규칙 ID변환설명
AR-1Q → H질문 유형(원인·영향·비교·탐색)을 분석하여 적절한 가설 형식(만약 A라면 B이다)을 자동 생성
AR-2H → E가설을 지지하는 증거(Pro-Evidence)와 반박하는 증거(Con-Evidence)를 자동 분류하여 수집 목록 생성
AR-3E → REQI 점수가 높은 증거부터 우선적으로 논리적 연결(추론 사슬)을 구성하고, 낮은 점수는 배제하거나 경고 표시
AR-4R → ReflectionReasoning Audit의 Confidence Score가 7 미만이면 자동으로 "Q 단계로 돌아가 질문 재정의"를 권고. 7 이상이면 최종 아웃풋 도출

📌 Module 1: 질문 설계하기 (Question Design)

QHER의 첫 단계. 좋은 질문이 좋은 사고의 출발점입니다.

① Experiment — 같은 주제, 다른 질문

당신이 해야 할 일: AI에게 같은 주제를 두 가지 방식으로 질문해보세요.

질문 A (열린 질문):

"AI가 교육에 어떤 영향을 미칠까?"

질문 B (구조화된 질문):

"AI가 교육에 미치는 영향을 다음 3가지 측면에서 분석해줘: ① 교사의 역할 변화, ② 학생의 학습 방식 변화, ③ 교육 기관의 운영 변화. 각 측면당 3가지 근거를 제시하고, 가장 큰 변화가 예상되는 순서로 정리해줘."

→ 어떤 질문이 더 깊이 있는 답변을 주었나요?

② Reflection — 생각해보기

  • 질문 A와 B의 답변 품질 차이는 무엇에서 발생했나요? (AI의 능력이 달라진 건가요, 아니면 질문이 달라진 건가요?)
  • 좋은 질문이 왜 좋은 답변을 만드는 걸까요?

③ Discovery — 방금 깨달은 것

"아, AI의 답변 품질은 AI의 능력이 아니라, 내가 던진 질문의 구조가 결정하는구나."

④ Rule — QHER 법칙

법칙내용
Rule Q1질문은 답변의 경계를 정의한다. 열린 질문은 넓지만 얕은 답변을, 구조화된 질문은 좁지만 깊은 답변을 만든다.
Rule Q2좋은 질문은 4가지 요소를 포함한다. ① 범위(Scope), ② 관점(Perspective), ③ 기준(Criteria), ④ 출력 형식(Format).
Rule Q3질문을 먼저 설계하고, AI를 나중에 실행하라. 뛰어난 질문은 뛰어난 AI보다 더 강력하다.

📌 Module 2: 가설 설계하기 (Hypothesis Design)

QHER의 두 번째 단계. 잠정적 답변(가설)을 구조화하고, 그 가설이 성립하는 조건을 명시합니다.

① Experiment — 가설 없이 질문하기

당신이 해야 할 일: 가설이 있을 때와 없을 때의 차이를 비교해보세요.

시나리오 A (가설 없음):

"우리 회사 생산성이 왜 낮을까?"

시나리오 B (가설 있음):

"우리 회사 생산성이 낮은 이유는 의사결정이 느리기 때문이라는 가설이 있어. 이 가설을 검증해줘. ① 이 가설을 지지하는 증거, ② 이 가설을 반박하는 증거, ③ 가설이 맞다면 어떤 개선책이 있을까?"

→ 시나리오 B에서 AI의 답변은 어떻게 달라졌나요?

② Reflection — 생각해보기

  • 가설이 있을 때, AI의 답변은 어떻게 달라졌나요?
  • 가설이 왜 사고를 더 깊게 만드는 걸까요?

③ Discovery — 방금 깨달은 것

"아, 좋은 질문만으로는 부족해. 나만의 잠정적 답변(가설)이 있어야 AI가 내 생각을 검증할 수 있구나."

④ Rule — QHER 법칙

법칙내용
Rule H1가설은 잠정적 답변이다. 확정된 정답이 아니라, 검증이 필요한 가정입니다.
Rule H2가설은 조건을 포함해야 한다. "만약 A라면 B이다" 형태로, 검증 가능해야 합니다.
Rule H3가설이 없으면 사고는 방황한다. 가설은 사고의 방향을 제시하고, 증거 수집을 안내합니다.

⑤ Practice — 가설 설계 템플릿

Step 1. AI_Work/reasoning/hypothesis_template.md 파일을 만드세요.

Step 2. 아래 내용을 채우세요.

```markdown
# QHER - 가설 설계 템플릿

## 1. 문제
[해결하려는 문제]

## 2. 가설 (Hypothesis)
[잠정적 답변. "만약 A라면 B일 것이다" 형식]

## 3. 가설의 조건 (Conditions)
- 조건 1: [가설이 성립하기 위해 필요한 전제]
- 조건 2: [가설이 성립하기 위해 필요한 전제]

## 4. 가설 검증 계획
- 지지 증거: [어떤 증거가 가설을 뒷받침할까?]
- 반박 증거: [어떤 증거가 가설을 무너뜨릴까?]
- 필요한 데이터: [검증에 필요한 구체적인 데이터]

## 5. 가설의 중요도 (Importance)
- 이 가설이 맞다면: [어떤 변화가 생길까?]
- 이 가설이 틀리다면: [어떤 변화가 생길까?]
```

Step 3. 현재 고민 중인 문제에 가설을 수립하고, AI에게 검증을 요청하세요.

⑥ Common Mistakes — 하지 마세요

❌ 하지 마세요✅ 이렇게 하세요
가설을 확정된 정답으로 착각한다.가설은 검증이 필요한 잠정적 답변이다.
가설 없이 바로 해결책을 찾는다.가설을 먼저 세우고, 검증한 후에 해결책을 찾아라.

⑦ Mini Project (20분) — 가설 세우기

목표: 현재 업무에서 맞닥뜨린 문제 1개에 대해 가설을 수립합니다.

  1. 현재 해결되지 않은 업무 문제 1개를 선택하세요.
  2. "만약 A라면 B일 것이다" 형식의 가설을 세우세요.
  3. 가설을 검증하기 위해 필요한 증거를 3가지 이상 나열하세요.
  4. AI에게 가설을 검증해달라고 요청하고, 결과를 기록하세요.

🎯 성공 기준: AI가 가설을 지지하거나 반박하는 구체적인 증거를 제시했다면 성공입니다!

⑧ Summary — 오늘의 정리

QHER 단계핵심 문장
H가설은 잠정적 답변이다
H"만약 A라면 B이다" 형식으로 검증 가능해야 한다
H가설이 사고의 방향을 제시한다

📌 Module 3: 증거 평가하기 — EQI 적용

QHER의 세 번째 단계. 가설을 검증하기 위해 증거를 수집하고, EQI(Evidence Quality Index)를 적용하여 품질을 평가합니다.

① Experiment — 증거의 질 차이

당신이 해야 할 일: 같은 주장에 대해 서로 다른 수준의 증거를 AI에게 제시해보세요.

증거 A (약한 증거):

"많은 사람들이 AI가 교육을 바꿀 거라고 말해."

증거 B (강한 증거):

"2024년 교육부 발표 자료에 따르면, AI 튜터를 사용한 학생들의 성적이 평균 15% 향상되었고, 특히 수학 과목에서 22%의 향상을 보였습니다. 이는 5,000명의 학생을 대상으로 한 무작위 대조군 실험 결과입니다."

→ 각 증거에 EQI 점수를 매겨보세요.

② Reflection — 생각해보기

  • 증거 A와 B의 EQI 점수는 각각 몇 점일까요?
  • EQI 점수 차이가 결론에 어떤 영향을 미칠까요?

③ Discovery — 방금 깨달은 것

"아, 증거의 품질을 숫자로 평가할 수 있구나. EQI가 있으면 어떤 증거를 믿을지 훨씬 객관적으로 판단할 수 있어."

④ Rule — QHER 법칙

법칙내용
Rule E1EQI = (출처×0.4) + (직접성×0.3) + (표본×0.2) + (재현성×0.1)
Rule E2EQI 4.0 이상은 강한 증거, 2.0 미만은 근거로 활용하지 마라.
Rule E3증거가 충돌할 때는 EQI가 높은 증거에 더 높은 가중치를 부여하라.

📌 Module 4: 추론 감사하기 (Reasoning Audit)

QHER의 네 번째 단계(R의 Inference). EQI로 평가된 증거를 논리적으로 연결하고, Reasoning Audit 체크리스트로 스스로 검증합니다.

  1. 자신의 결론(주장)을 정리하세요.
  2. Reasoning Audit 체크리스트(위에 정의된 6단계)를 실행하세요.
  3. Confidence Score를 산출하세요.

② Reflection — 생각해보기

  • Reasoning Audit를 실행하기 전과 후, 당신의 Confidence Score는 어떻게 달라졌나요?
  • 어떤 단계에서 가장 많은 약점이 발견되었나요?

③ Discovery — 방금 깨달은 것

"아, 내 추론에는 항상 숨겨진 가정과 누락된 증거가 있었구나. Audit가 없으면 절대 몰랐을 거야."

④ Rule — QHER 법칙

법칙내용
Rule R1Reasoning Audit는 6단계로 수행하라. ① 주장 점검, ② 가정 점검, ③ 증거 매핑, ④ 누락 증거, ⑤ 반박 고려, ⑥ 신뢰도 평가.
Rule R2Confidence Score 7 미만이면 재검토하라. 확신이 낮으면 Q 단계로 돌아가 질문을 재정의하세요.
Rule R3반박은 추론을 강화한다. Reasoning Audit에서 발견된 약점은 보강해야 할 부분입니다.

📌 Module 5: 판단과 종합하기 (Decision & Synthesis)

QHER의 네 번째 단계(R의 Decision/Synthesis). 불확실성을 고려한 판단(Decision)과 상충 의견의 통합(Synthesis)을 수행합니다.

① Experiment — 불확실성 하의 판단

당신이 해야 할 일: 아래 두 가지 선택지를 평가해보세요.

  • 선택지 A: 확실하게 100만 원을 받음
  • 선택지 B: 50% 확률로 300만 원을 받거나, 50% 확률로 0원을 받음

→ 어떤 선택을 하시겠습니까? (기대값은 둘 다 150만 원입니다)

② Reflection — 생각해보기

  • 단순히 기대값만으로 결정할 수 없는 이유는 무엇인가요? (리스크 회피, 손실 회피, 확실성 선호 등)
  • 당신의 결정에 영향을 미친 비금전적 요소는 무엇인가요?

③ Discovery — 방금 깨달은 것

"아, 좋은 판단은 숫자(기대값)만으로 하는 것이 아니라, 나의 리스크 성향과 맥락까지 고려하는 것이구나."

④ Rule — QHER 법칙

법칙내용
Rule R4판단 = 기대값 + 리스크 성향 + 맥락. 숫자만으로 결정하지 말고, 자신의 리스크 허용 범위와 상황을 고려하세요.
Rule R5Synthesis는 충돌하는 의견에서 새로운 모델을 만든다. 단순한 절충이 아니라, 더 높은 수준의 통합을 시도하세요.
Rule R6결정 후에는 Reflection으로 돌아가라. 결정이 옳았는지 검증하는 피드백 루프를 설계하세요.

④ Rule — QHER 법칙 (Engine)

법칙내용
Rule Loop1QHER Engine은 Q→H→E→R→Reflection의 자동 파이프라인이다.
Rule Loop2AR-1~4는 각 단계의 출력을 다음 단계의 입력으로 변환한다.
Rule Loop3Engine을 5회 이상 사용하면, 당신의 사고 패턴이 자동으로 최적화된다.
